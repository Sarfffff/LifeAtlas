# 旷野风格 UI 设计规范

本文档记录岁迹后续 UI 改版的选定方向。用户已确认选择第二张效果图方向作为 App 后续 UI 设计基准。

## 1. 设计关键词

核心表达：

```text
人生是旷野
```

关键词：

- 自由
- 开阔
- 温暖
- 探索
- 个人记忆
- 山野路径
- 地图坐标
- 旅行手账
- 照片回忆

不追求完全复刻参考图。参考图只提供温暖、亲和、吉祥物和人生地图的品牌气质；最终 UI 可以更自由、更有色彩。

## 2. 视觉方向

整体风格：

- 温暖奶油底色，减少纯白的冷感。
- 草地绿、天空蓝、日落橙和野花黄作为情绪色。
- 深青色用于标题、重点数字和核心按钮。
- 卡片保持圆润、柔和阴影和手账感。
- 页面可使用轻量插画元素，但不能影响信息密度和可用性。

建议色板：

```text
背景奶油白：#FFF7EA
纸张白：#FFFCF6
深青文字：#164F55
旷野绿：#6FA37A
草地浅绿：#BFDDB0
天空蓝：#9DCAE8
日落橙：#F4A261
野花黄：#F2C76E
珊瑚红：#E9786B
浅灰文本：#789193
```

## 3. 组件原则

### 顶部区域

- 首页可以使用“旷野路径”插画头图。
- 时间轴、地图、设置页不做沉重头图，保持轻量标题和功能入口。
- 标题文案要有情绪，但不要像营销页。

示例：

```text
岁迹
人生是旷野，今天从哪里出发？
```

### 卡片

- 记录卡片像旅行照片卡或手账卡片。
- 照片优先展示，但无照片时使用柔和插画占位。
- 卡片圆角建议 20dp 左右。
- 阴影轻，不使用厚重悬浮卡片。

### 按钮

- 主按钮使用深青或旷野绿。
- 重要操作按钮文案要短，例如“写新记忆”。
- 危险操作仍使用清晰红色，不为了风格牺牲安全提示。

### 标签

- 标签 chip 使用低饱和多色。
- 情绪、地点、日期可以使用不同 chip 风格区分。
- 不同颜色服务于识别，不做过度装饰。

### 底部导航

- 保持清晰、稳定。
- 可考虑中间突出“新增记录”按钮。
- 图标使用圆润线性风格。

## 4. 页面改版方向

### 首页

目标：打开 App 时有“从人生旷野继续出发”的感觉。

改版点：

- 顶部增加柔和旷野插画或色块。
- 增加一句情绪文案。
- 统计卡显示记录、地点、照片、标签。
- 主操作按钮改为“写一段新记忆”或“写新记忆”。
- 最近记录卡片增加照片、地点和标签优先级。

### 时间轴

目标：时间轴像一条穿过四季的路径。

改版点：

- 搜索栏更轻、更圆润。
- 标签筛选 chip 彩色化。
- 月份节点像路径坐标点。
- 记录卡片突出照片、日期、地点和心情。
- 空状态使用“还没有走到这里”的温柔表达。

### 地图

目标：地图页既是工具，也是人生足迹的总览。

改版点：

- 保留真实地图能力，不用纯插画替代。
- marker 样式更圆润，选中状态更明显。
- 底部详情卡像旅行照片卡。
- 未配置地图 Key 时使用更友好的旷野空状态。
- 后续可加入 marker 聚合。

### 设置

目标：设置页从“功能堆叠”变成清晰分组。

改版点：

- 分组为：账号与安全、数据备份、地图与记录、外观、关于岁迹。
- 每组使用浅色卡片背景。
- 加入 V2.0 账号入口占位，但明确当前本地优先。
- 备份导入导出按钮要更容易理解。

## 5. 吉祥物使用原则

吉祥物可以作为品牌陪伴元素，但不应该每个页面都强行出现。

建议使用位置：

- 首页顶部角落或空状态。
- 时间轴空状态。
- 备份成功提示。
- 新手引导或关于页。

避免：

- 在每个记录卡片重复出现。
- 遮挡主要信息。
- 过度幼态，影响长期工具使用感。

## 6. 实现顺序建议

第一阶段：低风险全局风格替换。

- 更新主题色。
- 更新背景色。
- 更新通用卡片、按钮、chip 风格。
- 更新底部导航选中态。

第二阶段：核心页面改版。

- 首页重做。
- 时间轴卡片和月份节点重做。
- 设置页分组重做。
- 地图页空状态和底部卡片重做。

第三阶段：插画和品牌资产。

- 生成或绘制首页旷野插画。
- 生成空状态吉祥物。
- 更新应用图标和商店物料。

## 7. 落地约束

- 先保证 Android Compose 可实现。
- 不使用过度复杂的位图背景，避免包体和性能压力。
- 插画资产独立管理，不能写死在业务组件中。
- 文案要温暖，但功能入口必须清楚。
- 不牺牲可读性、无障碍对比度和表单可用性。
