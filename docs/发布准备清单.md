# 发布准备清单

本文档记录岁迹进入 V1.0 候选版前需要完成的发布配置、隐私说明和构建检查。

## 1. 版本策略

当前版本：

```text
versionCode = 10
versionName = 1.0.0-rc1
```

正式发布 V1.0 时建议调整为：

```text
versionCode = 11
versionName = 1.0.0
```

规则：
- `versionCode` 每次发包必须递增。
- `versionName` 面向用户展示，按 `主版本.次版本.修订版本` 管理。
- 小范围自测使用 `1.0.0-rc1`，正式发布时再移除 `rc` 标记。

## 2. Release 签名配置

签名文件和密码不得提交到仓库。

在本机 `local.properties` 中配置：

```properties
lifeatlas.release.storeFile=keystore/lifeatlas-release.jks
lifeatlas.release.storePassword=你的 store 密码
lifeatlas.release.keyAlias=你的 key alias
lifeatlas.release.keyPassword=你的 key 密码
```

说明：
- `lifeatlas.release.storeFile` 是相对项目根目录的路径。
- `keystore/` 目录应保存在本机，不提交到 Git。
- 如果未配置签名信息，release 构建不会使用 release 签名配置。

## 3. 地图 Key 配置

高德 Key 仍放在本机 `local.properties`：

```properties
lifeatlas.amap.apiKey=你的高德 Key
```

发布前检查：
- 高德控制台已配置发布签名 SHA1。
- 包名与 `applicationId` 一致：`com.xiaoyin.lifeatlas`。
- 设置页“地图配置”显示 Key 已配置。
- 地图页可以正常展示真实地图。

## 4. 权限与隐私说明

当前权限：

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `ACCESS_COARSE_LOCATION`
- `ACCESS_FINE_LOCATION`

隐私说明必须包含：
- App 本地保存用户记录、照片引用、标签、地点和备份数据。
- 默认不强制登录，不自动上传个人记录。
- 地图能力使用高德地图 SDK。
- 用户点击“使用当前位置”时才请求前台定位权限。
- 不申请后台定位权限。
- 照片选择使用 Android Photo Picker，不申请完整相册读取权限。
- 完整备份包会导出结构化数据和 App 私有媒体缓存，文件由用户自行保存。


## 6. 发布前阻断项

以下问题未解决前不建议发布：

- Debug 构建失败
- Release 构建失败
- Room Migration 测试失败
- JSON 导入导出失败
- 完整备份包导入后媒体缓存无法显示
- 地图 Key 未配置时崩溃
- 定位权限拒绝后无法手动选点
- 删除记录失败时没有提示
- 隐私说明未覆盖地图、定位、照片和备份包
