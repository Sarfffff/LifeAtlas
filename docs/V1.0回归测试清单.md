# V1.0 回归测试清单

本文档用于第一版发布前的手动回归测试。每次准备发布或大模块合并后，按清单完整走一遍。

## 1. 测试环境

- Android Studio 打开项目目录：`E:\岁迹`
- 测试版本：`1.0.0`
- 构建类型：Debug，发布前补测 Release
- 测试设备：Android 模拟器或真机
- 建议至少测试一次无地图 Key 环境和一次已配置高德 Key 环境

## 3. 首页与时间轴

- 打开 App，首页可以显示统计数据
- 首页最近记录列表可以正常展示
- 切换到时间轴页面
- 无记录时显示空状态
- 有记录时按月份分组显示
- 时间轴搜索可以按标题、正文、地点、心情过滤
- 清空搜索后列表恢复
- 标签筛选可以过滤记录

## 4. 新增记录

- 新增一条只包含标题和正文的记录
- 新增一条包含日期、地点、心情、重要程度的记录
- 新增一条包含标签的记录
- 新增一条包含照片的记录
- 新增一条包含经纬度的记录
- 保存后可以在时间轴看到新记录
- 首页统计数量随记录变化

## 5. 编辑与删除

- 从详情页进入编辑页
- 修改标题、正文、日期、地点、心情和重要程度
- 新增、删除或替换照片
- 修改标签
- 保存后详情页和时间轴同步更新
- 删除记录前出现二次确认
- 确认删除后记录从时间轴和地图中消失

## 6. 照片能力

- Photo Picker 可以选择多张照片
- 新增记录保存后详情页可看到照片
- 时间轴卡片优先显示首张照片
- 照片缩略图可以正常显示
- 照片加载失败时显示占位提示
- 编辑记录移除照片后不再显示该照片

## 7. 地图能力

- 未配置地图 Key 时，地图页显示配置提示
- 已配置地图 Key 时，地图页显示真实地图
- 带坐标记录显示 marker
- 点击 marker 可以进入对应详情页
- 新增记录页可以打开地图选点
- 编辑记录页可以打开地图选点
- 地图选点后经纬度回填表单
- 地址反查成功时地点自动回填
- 使用当前位置时会请求前台定位权限
- 拒绝定位权限后仍可手动选点

## 8. 数据导出与导入

- 设置页 JSON 导出可以生成 `lifeatlas_export.json`
- JSON 导入前显示备份摘要
- 确认导入后记录、照片引用、标签恢复
- 完整备份包可以生成 `lifeatlas_backup.zip`
- zip 内包含 `lifeatlas_export.json`
- zip 内包含 `backup_manifest.json`
- zip 内包含可用媒体缓存目录
- zip 导入前显示备份摘要和媒体文件数量
- 确认 zip 导入后记录和媒体缓存恢复

## 9. 设置页

- 本地优先开关可以切换并持久化
- 标签管理入口可以进入
- 地图配置状态显示正确
- 版本信息显示版本号、构建号和包名

## 10. 发布前阻断项

出现以下情况时，不建议发布：
- App 启动崩溃
- 新增记录无法保存
- 详情页无法打开
- 编辑保存后数据丢失
- JSON 导入导出失败
- 完整备份包导入后照片缓存无法显示
- 地图 Key 未配置时出现崩溃
- 权限拒绝后功能无法继续使用

## 11. 当前已知边界

- 暂未接入云同步
- 暂未接入登录体系
- 暂未恢复旧设备系统相册原图，只恢复备份包内 App 私有媒体缓存
- 自动化测试覆盖仍需后续补齐
