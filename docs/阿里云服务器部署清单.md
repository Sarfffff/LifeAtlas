# 阿里云服务器部署清单

## 当前服务器信息

- 云厂商：阿里云
- 地域：华中 1（武汉）
- 系统：Ubuntu 22.04
- 规格：2 vCPU / 2 GB / 40 GB
- 公网 IP：`47.122.124.84`
- 私网 IP：`172.18.17.119`
- 后端服务端口：`8080`
- 域名：`lifeatlas.cn`
- 后端接口域名：`api.lifeatlas.cn`

## 域名解析

域名完成实名认证后，在阿里云 DNS 解析中添加：

```text
主机记录：api
记录类型：A
记录值：47.122.124.84
TTL：默认
```

如果后续要做个人网页，再添加：

```text
主机记录：@
记录类型：A
记录值：47.122.124.84

主机记录：www
记录类型：A
记录值：47.122.124.84
```

## App 后端地址

域名解析生效并配置 HTTPS 后，App 使用：

```properties
lifeatlas.auth.provider=backend
lifeatlas.auth.baseUrl=https://api.lifeatlas.cn
```

当前域名已确定为 `lifeatlas.cn`，App 后端地址固定为 `https://api.lifeatlas.cn`。

## 服务器安全组

在阿里云服务器安全组中放行：

```text
22/tcp    SSH 管理
80/tcp    HTTP，用于证书申请和跳转
443/tcp   HTTPS，App 正式访问
```

后端内部端口 `8080` 不建议公网放行，由 Nginx 反向代理到本机。

## 服务器初始化命令

```bash
sudo apt update
sudo apt install -y nginx git curl
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

## 部署目录

```bash
sudo mkdir -p /opt/lifeatlas
sudo chown -R $USER:$USER /opt/lifeatlas
```

将仓库中的 `server/` 上传到：

```text
/opt/lifeatlas/server
```

然后：

```bash
cd /opt/lifeatlas/server
cp .env.example .env
npm install --omit=dev
npm start
```

浏览器或命令行验证：

```bash
curl http://127.0.0.1:8080/health
```

应返回：

```json
{"ok":true,"service":"LifeAtlas Auth Server"}
```

## Nginx 配置

复制模板：

```bash
sudo cp /opt/lifeatlas/server/deploy/nginx-lifeatlas.conf.example /etc/nginx/sites-available/lifeatlas-auth
sudo nano /etc/nginx/sites-available/lifeatlas-auth
```

模板已使用 `api.lifeatlas.cn`，如未来更换域名再修改。

启用站点：

```bash
sudo ln -s /etc/nginx/sites-available/lifeatlas-auth /etc/nginx/sites-enabled/lifeatlas-auth
sudo nginx -t
sudo systemctl reload nginx
```

## 后台运行

复制 systemd 模板：

```bash
sudo cp /opt/lifeatlas/server/deploy/lifeatlas-auth.service.example /etc/systemd/system/lifeatlas-auth.service
sudo systemctl daemon-reload
sudo systemctl enable lifeatlas-auth
sudo systemctl start lifeatlas-auth
sudo systemctl status lifeatlas-auth
```

查看日志：

```bash
journalctl -u lifeatlas-auth -f
```

## HTTPS

域名备案与解析完成后，推荐使用 Certbot：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d api.lifeatlas.cn
```

证书完成后确认 App 的 `lifeatlas.auth.baseUrl` 为 `https://api.lifeatlas.cn`。

## 阿里云邮件服务

开通阿里云邮件推送 / DirectMail 后，更新服务器 `.env`：

```env
SMTP_HOST=smtpdm.aliyun.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=你的发信地址
SMTP_PASS=你的SMTP密码
SMTP_FROM="岁迹 <你的发信地址>"
```

然后重启：

```bash
sudo systemctl restart lifeatlas-auth
```

## 2026-06-26 登录注册专项排查

当前从本机访问 `https://api.lifeatlas.cn/health` 会出现 `Recv failure: Connection was reset`，这不是 App 表单或 Firebase 问题，而是域名入口被阿里云 ICP 备案合规拦截：

- `http://api.lifeatlas.cn/health` 返回 `Non-compliance ICP Filing`。
- `https://api.lifeatlas.cn/health` 在 TLS/HTTPS 层直接连接重置。
- `https://47.122.124.84/health -k` 可以访问到后端，但正常证书校验会失败，因为证书签发给域名，不签发给 IP。
- `http://47.122.124.84/health` 当前返回 Nginx 404，说明 HTTP IP 默认站点没有代理到后端。

建议处理顺序：

1. 正式方案：完成 `lifeatlas.cn` 备案，并确保 `api.lifeatlas.cn` 的 HTTPS 证书和 Nginx `server_name` 配置正确。
2. 临时测试方案：在 Nginx 为公网 IP 或临时测试域名配置 HTTP 反向代理，再只在 Debug 包中允许明文 HTTP。
3. 不建议在正式包使用 IP + 跳过证书校验方案，Android 正式网络请求也不应跳过证书校验。
