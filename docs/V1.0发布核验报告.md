# V1.0 发布核验报告

日期：2026-06-26

本文档记录当前 `1.0.0` 体验版在完成六步补齐计划后的发布前状态。

## 1. 六步补齐计划状态

| 步骤 | 状态 | 说明 |
| --- | --- | --- |
| 记录偏好持久化 | 已完成 | 默认心情、默认标签、照片保存策略已持久化，并自动带入新增记录页。 |
| 收藏中心 | 已完成 | 设置页可进入收藏中心，支持查看收藏记录、进入详情和取消收藏。 |
| 城市详情页 | 已完成 | 地图底部卡片可进入城市详情，按城市/地点聚合点亮记录。 |
| 账号安全完善 | 已完成 | 登录状态页展示账号安全信息，支持重置密码入口、退出确认和清除本地账号确认。 |
| 云同步基础设置 | 已完成基础层 | 已有云同步准备开关、Firebase 状态、最近检查时间和备份包迁移入口；暂不自动上传数据。 |
| 发布准备 | 已完成当前候选版核验 | 已补充发布核验报告，并重新构建 Debug/Release 包。 |

## 3. 当前可体验重点

1. 首页、时间轴、地图、设置页的旷野风格 UI。
2. 新增、编辑、删除记录，照片导入和记录详情。
3. 地图点亮地点、城市筛选、城市详情、收藏和分享。
4. 收藏中心集中查看收藏记录。
5. 设置页资料修改、记录偏好、备份导出/恢复、云同步准备状态。
6. Firebase 邮箱登录/注册能力和本地账号兜底能力。

## 4. 正式上架前阻断项

以下事项未完成前，不建议发布到应用市场：

- 生成正式 release keystore，并妥善备份。
- 在高德控制台配置发布签名 SHA1。
- 使用 release keystore 重新构建正式签名 APK/AAB。
- 完成至少一轮真机回归测试并记录结果。
- 审阅并定稿隐私政策，补充开发者联系方式、第三方 SDK 清单和生效日期。
- 确认 Firebase 邮箱登录在目标网络环境下可用。
- 确认地图、定位、照片选择、备份导入导出在目标机型上可用。

## 5. 后续建议

个人使用阶段可以先继续使用 Debug 体验包和完整备份包迁移。若准备上线，下一步优先做：

1. 真机回归测试记录。
2. release keystore 与发布 SHA1 配置。
3. 隐私政策正式版。
4. 应用市场素材：512 图标、截图、简介、权限说明。
