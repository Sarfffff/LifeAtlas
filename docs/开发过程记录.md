#### 问题 1：项目路径包含中文导致 Android Gradle Plugin 拒绝构建

报错摘要：

```text

```

处理方式：

在 `gradle.properties` 中增加：

```properties
android.overridePathCheck=true
```

原因：

当前项目路径为 `E:\岁迹`，在 Windows 上 Android Gradle Plugin 默认会阻止非 ASCII 路径。

#### 问题 2：本机没有全局 gradle 命令

报错摘要：


处理方式：

### 遇到的问题与处理

#### 问题 1：`kapt` 插件版本解析冲突

报错摘要：

```text
Error resolving plugin [id: 'org.jetbrains.kotlin.kapt', version: '2.0.21']
The request for this plugin could not be satisfied because the plugin is already on the classpath with an unknown version
```

原因：

项目已经通过 Kotlin Android 插件加载了 Kotlin 插件，版本目录中再次以带版本 alias 的方式声明 `org.jetbrains.kotlin.kapt`，导致 Gradle 无法确认兼容性。

处理方式：

从 `gradle/libs.versions.toml` 删除 `kotlin-kapt` 插件 alias，在 `app/build.gradle.kts` 中使用无版本插件声明：

```kotlin
id("kotlin-kapt")
```

#### 问题 2：Kapt 对 Kotlin 2.0 的提示

构建警告：

```text
Kapt currently doesn't support language version 2.0+. Falling back to 1.9.
```

当前处理：

这是警告，不影响构建成功。后续可考虑将 Room 注解处理从 `kapt` 迁移到 `ksp`，减少 Kotlin 2.0 下的兼容警告。

接入照片导入能力，让用户在新增记录时可以通过 Android Photo Picker 选择多张照片，并在记录详情页查看这些照片。

### 技术取舍

本阶段选择：

```text
保存 Photo Picker 返回的图片 URI，不复制原图，不做压缩存储
```

原因：

- 实现难度最低，能最快打通“记录关联照片”的核心闭环。
- 不需要申请完整相册读取权限，符合隐私优先原则。
- 不会额外占用 App 私有存储空间。
- 适合 V0.1 验证功能可用性。

暂不选择“压缩复制到 App 私有目录”的原因：

- 需要处理图片解码、压缩质量、后台任务、缩略图缓存和存储清理。
- 需要更多异常处理，例如大图 OOM、压缩失败、磁盘空间不足。
- 会拉长 V0.1 周期，不利于先验证产品闭环。

后续计划：

- V0.2 或 V1.0 再增加缩略图和压缩图缓存。
- 长期备份或导出时，再考虑将图片复制到完整备份包中。

### 操作

## 2026-06-24 V0.1 第十四步：时间轴搜索

### 目标

增加时间轴内的快速搜索能力，让用户可以在记录数量变多后，通过关键词找回人生节点。

本阶段搜索范围：

```text
标题
正文
地点
心情
```

搜索会和已有标签筛选叠加使用。

### 技术取舍

本阶段选择在 `TimelineViewModel` 中对当前记录流做内存过滤，而不是直接新增数据库全文搜索。

原因：

- V0.1 数据量较小，内存过滤实现简单、响应直接。
- 不需要改变数据库结构，也不需要引入 FTS 表。
- 可以和现有标签筛选流组合，减少 DAO 复杂度。
- 后续数据量变大后，再升级为 Room FTS 或分页搜索。

### 操作

1. `TimelineUiState` 增加 `searchQuery`。
2. `TimelineViewModel` 增加搜索关键词状态。
3. 使用 `combine` 将记录流、标签筛选和搜索关键词组合。
4. 增加 `filterByQuery()`，匹配标题、正文、地点、心情。
5. 时间轴页面增加搜索输入框。
6. 搜索框支持一键清空。
7. 根据搜索状态和标签状态展示不同空状态文案。
8. 将 Android Studio 生成的 `.idea/` 加入 `.gitignore`，避免误提交本机 IDE 配置。
