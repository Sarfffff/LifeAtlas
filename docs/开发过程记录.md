## 2026-06-25 本地账号安全限制

### 目标

在真实邮箱验证和云端账号接入前，先补齐本地登录注册的基础安全策略，降低误操作和恶意尝试风险。

### 操作

1. 本地注册不再覆盖已有账号，已有账号时提示先登录或清除本地账号。
2. 登录失败次数持久化记录，连续失败 5 次后进入 10 分钟冷却。

5. 账号页补充安全策略说明、失败次数提示和冷却剩余时间提示。

## 2026-06-25 Firebase 真实邮箱登录准备

### 目标

把登录注册从本地占位推进到真实邮箱能力：注册后发送邮箱验证邮件，忘记密码时发送重置邮件，QQ 邮箱可作为收件邮箱使用。

### 操作

1. 增加 Firebase Auth 依赖。
2. 增加 Google Services Gradle 插件声明。
3. App 模块做条件接入：只有 `app/google-services.json` 存在时才应用 Google Services 插件，避免当前缺配置文件时无法构建。
4. `AuthRepository` 在 Firebase 已配置时优先使用 Firebase Authentication。
5. 注册成功后调用 Firebase 发送验证邮件。
6. 忘记密码调用 Firebase 发送密码重置邮件。
7. 登录成功后同步邮箱、验证状态和登录状态到本地 DataStore。
8. 账号页增加“重新发送验证邮件”和“我已验证，刷新状态”。

### 需要用户提供

1. Firebase 控制台创建项目。
2. 添加 Android App：

```text
包名：com.xiaoyin.lifeatlas
```

3. 下载 `google-services.json`，放到：

```text
E:\岁迹\app\google-services.json
```

4. 在 Firebase Authentication 中启用：

```text
Email/Password
```

5. 如果后续要正式发布，需要补充发布签名 SHA-1/SHA-256。

不建议把 QQ 邮箱 SMTP 授权码写进 Android App，因为客户端会被反编译，邮箱授权码存在泄露风险。Firebase Auth 更适合作为真实邮箱验证入口。

#### 问题 1：`kapt` 插件版本解析冲突

```text
Error resolving plugin [id: 'org.jetbrains.kotlin.kapt', version: '2.0.21']

```

原因：

项目已经通过 Kotlin Android 插件加载了 Kotlin 插件，版本目录中再次以带版本 alias 的方式声明 `org.jetbrains.kotlin.kapt`，导致 Gradle 无法确认兼容性。

处理方式：

从 `gradle/libs.versions.toml` 删除 `kotlin-kapt` 插件 alias，在 `app/build.gradle.kts` 中使用无版本插件声明：

```kotlin
id("kotlin-kapt")
```

#### 问题 2：Kapt 对 Kotlin 2.0 的提示

构建警告：

```text
Kapt currently doesn't support language version 2.0+. Falling back to 1.9.
```

当前处理：

这是警告，不影响构建成功。后续可考虑将 Room 注解处理从 `kapt` 迁移到 `ksp`，减少 Kotlin 2.0 下的兼容警告。

## 2026-06-24 V0.1 第三十九步：照片私有缩略图缓存

### 目标

- 缩略图体积更小，适合当前时间轴和详情页预览场景。

- 原始 URI 继续保留，后续仍可扩展为完整媒体备份包。

5. 时间轴首图和详情页照片优先展示 `Photo.displayUri`。

7. 编辑记录替换照片时清理旧缩略图文件。
8. 删除记录时清理关联缩略图文件。

## 本轮新增：首次启动登录门禁

在本地账号骨架基础上，本轮继续补齐登录注册的实际进入流程：

- App 首次启动时，如果未登录且未选择跳过，会自动进入账号页。
- 账号页不显示底部导航，避免未进入 App 前误操作其它页面。

- 从设置页进入账号页时，继续按钮会返回上一页。
- 登录页新增“忘记密码”占位入口，后续接入 Firebase 后可替换为真实邮件重置密码。
