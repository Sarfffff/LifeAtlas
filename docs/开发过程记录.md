### 遇到的问题与处理

#### 问题 1：项目路径包含中文导致 Android Gradle Plugin 拒绝构建

报错摘要：

```text
Your project path contains non-ASCII characters.
```

处理方式：

在 `gradle.properties` 中增加：

```properties
android.overridePathCheck=true
```

原因：

当前项目路径为 `E:\岁迹`，在 Windows 上 Android Gradle Plugin 默认会阻止非 ASCII 路径。

#### 问题 2：本机没有全局 gradle 命令

处理方式：

使用本机 Gradle Wrapper 缓存中的 Gradle：

```text
C:\Users\86156\.gradle\wrapper\dists\gradle-8.11.1-bin\...\gradle-8.11.1\bin\gradle.bat
```

#### 问题 3：`gradlew.bat` 首次运行尝试下载 Gradle 超时

报错摘要：

```text
Downloading https://services.gradle.org/distributions/gradle-8.11.1-bin.zip failed: timeout
```

处理方式：

暂时用本机已解压 Gradle 编译。后续网络正常后，`gradlew.bat` 可自动下载对应版本。

#### 问题 4：找不到 Android SDK

报错摘要：

```text
SDK location not found
```

处理方式：

创建本机私有 `local.properties`：

```properties
sdk.dir=C:/Users/86156/AppData/Local/Android/Sdk
```

并在 `.gitignore` 中忽略：

```gitignore
local.properties
```

### 遇到的问题与处理

#### 问题 1：`kapt` 插件版本解析冲突

报错摘要：

```text
Error resolving plugin [id: 'org.jetbrains.kotlin.kapt', version: '2.0.21']
The request for this plugin could not be satisfied because the plugin is already on the classpath with an unknown version
```

原因：

项目已经通过 Kotlin Android 插件加载了 Kotlin 插件，版本目录中再次以带版本 alias 的方式声明 `org.jetbrains.kotlin.kapt`，导致 Gradle 无法确认兼容性。

处理方式：

从 `gradle/libs.versions.toml` 删除 `kotlin-kapt` 插件 alias，在 `app/build.gradle.kts` 中使用无版本插件声明：

```kotlin
id("kotlin-kapt")
```

#### 问题 2：Kapt 对 Kotlin 2.0 的提示

构建警告：

```text
Kapt currently doesn't support language version 2.0+. Falling back to 1.9.
```

当前处理：

这是警告，不影响构建成功。后续可考虑将 Room 注解处理从 `kapt` 迁移到 `ksp`，减少 Kotlin 2.0 下的兼容警告。
