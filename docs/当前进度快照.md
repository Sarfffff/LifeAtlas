## 当前阶段

项目处于 V0.1 本地记录闭环开发阶段。

V0.1 的目标是先完成一个不依赖登录、不依赖云端、以 Room 本地数据库为核心的最小可用版本：

```text
新增记录 -> 本地保存 -> 时间轴查看 -> 详情查看 -> 编辑/删除 -> 备份导入导出
```

## 已知限制

- 地图页已有真实地图容器和记录 marker，但 marker 点击暂不跳转详情。
- 地图选点入口目前是占位提示，还不能真实选择坐标。
- 经纬度目前仍需要手动填写，还没有当前位置获取。
- JSON 导入没有差异预览。
- JSON 导入按原 ID 恢复，同 ID 本地数据会被覆盖。
- 照片备份目前只保存 URI 引用，不打包照片文件本体。
- 还没有系统化自动化测试和真机完整回归测试。

## 下一步建议

建议下一个独立功能模块：

```text
地图 marker 点击查看详情
```

原因：

- 地图 marker 已经可以展示。
- 下一步应让用户点击 marker 后进入对应记录详情。
- 这样地图页可以从“展示点位”进一步变成可探索入口。

推荐实现边界：

- marker 绑定记录 ID。
- 点击 marker 时回调到地图页。
- 地图页通过导航进入记录详情。
- 不申请定位权限。
