## 当前阶段

项目处于 V0.1 本地记录闭环开发阶段。

V0.1 的目标是先完成一个不依赖登录、不依赖云端、以 Room 本地数据库为核心的最小可用版本：

```text
新增记录 -> 本地保存 -> 时间轴查看 -> 详情查看 -> 编辑/删除 -> 备份导入导出
```

## 已知限制

- 编辑记录时暂不支持照片增删管理。
- 地图页目前是点位列表，还没有接入真实地图 SDK。
- 经纬度目前需要手动填写，还没有地图选点和当前位置获取。
- JSON 导入没有差异预览。
- JSON 导入按原 ID 恢复，同 ID 本地数据会被覆盖。
- 照片备份目前只保存 URI 引用，不打包照片文件本体。
- 设置页“本地优先”开关目前只是 UI 状态，尚未持久化。
- 还没有系统化自动化测试和真机完整回归测试。

## 下一步建议

建议下一个独立功能模块：

```text
编辑记录时支持照片增删管理
```

原因：

- 这是当前记录闭环中最明显的缺口。
- 新增时可以选照片，但编辑时不能调整照片。
- 做完后，文字、日期、标签、坐标、照片的编辑能力才完整。

推荐实现边界：

- 编辑页展示当前照片。
- 支持新增照片。
- 支持移除照片引用。
- 保存编辑时同步更新照片列表。
- 不做照片压缩和本体备份，保持模块边界清晰。
