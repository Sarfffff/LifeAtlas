# 个人网站交接摘要

记录日期：2026-06-27

本文用于在新的 Codex 对话或新的项目目录中开发个人博客/个人网站时，快速理解岁迹 App 的背景。不要把网站源码放进当前 Android App 仓库。

## 岁迹项目概览

- 中文名：岁迹
- 副标题：我的人生地图
- 英文名 / 仓库名：LifeAtlas
- 定位：本地优先的人生记录 App，用时间轴、地图、照片和文字记录个人记忆。
- 主题气质：人生是旷野、温暖自然、旅行感、地图点亮、个人记忆。

## 当前 App 能力

- 首页：照片轮播、统计卡、写一段新记忆、最近记忆。
- 时间轴：搜索、分类筛选、收藏筛选、星级重要程度。
- 地图：真实地图、城市筛选、搜索、点亮新城市、温馨点亮页、城市详情。
- 记录：新增、编辑、详情、照片导入、地图选点、收藏、分享、删除。
- 设置：头像昵称签名、记录偏好、标签管理、隐私权限说明。
- 数据：Room 本地持久化、完整备份包导出/恢复、JSON 导出/恢复。
- 账号：本地账号和国内后端邮箱验证预留；真实邮件闭环等待备案和邮件服务稳定。

## 当前技术栈

- Android：Kotlin、Jetpack Compose、Room、DataStore。
- 地图：高德地图 SDK。
- 图片：Coil。
- 后端雏形：Node.js 轻量认证服务，目录为 `server/`。
- 域名：`lifeatlas.cn`。
- API 子域名：`api.lifeatlas.cn`。

## 个人网站建议方向

- 网站作为个人主页 + 项目展示页，不直接承载 App 数据。
- 可展示：
  - 个人介绍。
  - 岁迹项目介绍。
  - App 截图和功能亮点。
  - 开发日志 / 博客文章。
  - APK 下载入口或 GitHub 链接。
  - 后续隐私政策、用户协议、备案信息页。
- 不建议在第一版网站中做：
  - 读取用户 App 私有数据。
  - Web 端登录同步。
  - 在线管理 App 记录。

## 推荐目录隔离

建议新建独立目录，例如：

```text
E:\个人网站
```

或者：

```text
E:\LifeAtlasSite
```

不要放在：

```text
E:\岁迹
```

## 推荐网站技术栈

如果想快速上线、维护简单：

- Next.js + TypeScript + Tailwind CSS。
- 静态内容优先，博客文章用 Markdown / MDX。
- 后续可部署到阿里云服务器、Vercel、Netlify 或 Cloudflare Pages。

如果想完全国内访问优先：

- Next.js 静态导出。
- Nginx 部署到阿里云服务器。
- 使用 `www.lifeatlas.cn` 或 `blog.lifeatlas.cn`。

## 和 App 的嵌入方式

短期推荐“展示型嵌入”：

- 网站展示 App 介绍、截图、功能列表、开发进度和 APK 下载入口。
- 网站链接到 GitHub 仓库。
- App 设置页或关于页后续可跳转到网站。

中期可做“文档型嵌入”：

- 网站提供隐私政策、用户协议、版本日志、帮助文档。
- App 内通过外部浏览器打开这些页面。

长期等云同步完成后再考虑“数据型嵌入”：

- Web 端登录。
- 查看时间轴。
- 查看人生地图。
- 管理照片和记录。

请先读取我提供的《个人网站交接摘要》，然后帮我规划网站信息架构、技术选型和项目框架。
