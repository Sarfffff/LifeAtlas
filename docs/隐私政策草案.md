# 隐私政策草案

本文档是岁迹 V1.0 正式体验版的隐私政策草案，用于后续个人使用、内测或正式上架前审阅。正式发布前应结合实际上架平台要求、第三方 SDK 文档和应用内展示方式进行最终确认。

## 1. 产品说明

岁迹是一款本地优先的人生记录 App，用于帮助用户记录文字、照片、时间、地点、标签和人生节点，并通过时间轴和地图进行回看。

## 2. 我们处理的数据

在当前版本中，App 可能在用户设备本地保存以下数据：

- 用户创建的记录标题、正文、发生日期、地点、经纬度、心情和重要程度。
- 用户为记录添加的标签，以及记录和标签之间的关联关系。
- 用户通过 Android Photo Picker 选择的照片引用。
- App 为了稳定展示照片而生成的私有媒体缓存或缩略图。
- 用户主动导出的 JSON 备份文件或完整备份包。
- 本地设置项，例如本地优先开关状态。

当前版本默认不要求登录，不提供云同步，不会自动把用户的人生记录上传到服务器。

## 3. 照片访问

App 使用 Android Photo Picker 让用户选择照片。该方式由系统提供，用户只授权自己选择的照片，App 不申请完整相册读取权限。

为了避免原始照片 URI 长期失效，App 可能在应用私有目录中生成用于展示的媒体缓存。该缓存只用于 App 内显示和备份恢复。

## 4. 位置信息

App 支持记录地点和经纬度，用于在地图上展示人生记录。

当用户点击“使用当前位置”时，App 会请求前台定位权限。App 不申请后台定位权限，也不会在后台持续追踪位置。

用户也可以不授权定位权限，改为在地图上手动选点。

## 5. 第三方 SDK 与网络

地图能力使用高德地图 SDK。地图展示、定位、地址反查等能力可能需要网络连接，并可能由高德按照其 SDK 规则处理必要的设备、网络、定位或地图服务相关信息。

正式发布前，需要在应用市场隐私合规材料中补充高德地图 SDK 的名称、用途、处理信息类型和第三方隐私政策链接，并按照高德官方文档完成合规配置。

## 6. 数据导出与导入

用户可以在设置页主动导出 JSON 备份文件或完整备份包。

完整备份包可能包含结构化记录数据、备份清单和 App 私有媒体缓存。导出的文件由用户自行保存、分享或删除，请妥善保管。

用户导入备份前，App 会展示备份摘要，用户确认后才会执行导入。

## 7. 权限说明

当前版本可能使用以下权限：

- `INTERNET`：用于地图展示、地址反查等网络能力。
- `ACCESS_NETWORK_STATE`：用于判断网络状态，辅助地图相关能力。
- `ACCESS_COARSE_LOCATION`：用于用户主动选择当前位置时获取大致位置。
- `ACCESS_FINE_LOCATION`：用于用户主动选择当前位置时获取较精确位置。

App 不申请后台定位权限。

## 8. 数据删除

用户可以在 App 内删除单条记录。删除记录后，该记录的文字、地点、标签关联和照片引用不再出现在 App 中。

如果用户希望彻底清除本机所有 App 数据，可以通过系统设置中的应用信息清除存储数据，或卸载 App。

## 9. 未成年人使用

当前版本面向个人记录场景，不专门面向未成年人提供服务。未成年人使用前应取得监护人同意。

## 10. 联系与更新

本隐私政策会随着功能变化持续更新。正式发布前应补充开发者联系方式、第三方 SDK 清单和隐私政策生效日期。
