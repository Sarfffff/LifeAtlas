# 阿里云邮箱验证接入方案

## 目标

岁迹面向国内用户时，不再把 Firebase 作为默认注册登录链路，避免因为 Google 服务网络不可达导致注册一直转圈。

目标链路：

```text
App -> 自建认证后端 -> 阿里云邮件服务 -> 用户邮箱
```

## 购买建议

优先购买：

- 阿里云轻量应用服务器
- 系统镜像：Ubuntu 22.04 LTS
- 规格：2 vCPU / 2 GB / 40 GB
- 带宽：3M 起步即可；如果页面显示 200 Mbps 峰值公网带宽，也足够早期使用
- 地域：优先中国内地；如果域名备案暂时不方便，可以先用中国香港或新加坡做测试

暂不建议购买：

- OpenClaw、Hermes Agent 等应用镜像。这类镜像偏现成应用，不适合作为岁迹后端基础环境。
- 短信服务。短信验证码成本更高，先完成邮箱验证更合适。

后续需要：

- 域名：`lifeatlas.cn`
- 后端接口域名：`https://api.lifeatlas.cn`
- SSL 证书，可使用阿里云免费证书或 Let's Encrypt
- 阿里云邮件推送 / DirectMail，用于验证码邮件

## App 配置

默认情况下，App 会使用本地账号，不访问 Firebase。

接入国内后端时，在 `local.properties` 中加入：

```properties
lifeatlas.auth.provider=backend
lifeatlas.auth.baseUrl=https://api.lifeatlas.cn
```

只有显式需要 Firebase 时才写：

```properties
lifeatlas.auth.provider=firebase
```

## 服务端目录

当前已预留 `server/` 目录：

- `server/src/index.js`：认证后端入口
- `server/.env.example`：部署配置模板
- `server/deploy/nginx-lifeatlas.conf.example`：Nginx 反向代理模板
- `server/deploy/lifeatlas-auth.service.example`：systemd 后台服务模板
- `server/package.json`：Node 运行依赖
- `server/README.md`：接口和运行说明

当前服务端已包含：

- 邮箱密码注册
- 邮箱密码登录
- 发送邮箱验证邮件
- 邮箱验证落地页
- 密码重置邮件预留接口
- 简单注册频率限制
- JSON 文件持久化用户数据

## 部署后要做的事

1. 在服务器安装 Node.js 20 LTS。
2. 上传 `server/` 目录。
3. 复制 `.env.example` 为 `.env`。
4. 填写阿里云邮件 SMTP 配置。
5. 使用 `npm install` 安装依赖。
6. 使用 `npm start` 验证 `/health`。
7. 配置 Nginx 和 HTTPS。
8. 在 App 的 `local.properties` 写入 `lifeatlas.auth.baseUrl`。
9. 重新构建 APK。
