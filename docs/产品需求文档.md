# 岁迹 LifeAtlas 产品需求文档

## 1. 产品概述

### 1.1 产品名称

- 中文名：岁迹
- 完整命名：岁迹 | 我的人生地图
- 英文项目名：LifeAtlas

### 1.2 产品定位

岁迹是一款基于地图和时间轴的人生记录 App。用户可以把人生经历以地点、时间、照片、文字、人物、心情、标签等形式记录下来，最终形成一张属于自己的“人生地图”。

### 1.3 核心价值

用户可以回看自己去过哪里、发生过什么、拍过什么照片、写下过什么感受。产品强调长期记录、私密保存、可回看、可导出，而不是社交曝光或实时轨迹追踪。

### 1.4 目标用户

- 想长期记录生活、旅行、成长节点的人
- 喜欢用照片、文字和地点整理回忆的人
- 重视隐私，不希望个人经历强制上传云端的人
- 希望未来能生成年度回顾、人生地图、城市足迹的人

## 2. MVP 范围

### 2.1 第一版目标

第一版只做最核心记录闭环：

1. 新增一条人生记录
2. 保存标题、正文、时间、地点等基础信息
3. 在时间轴中展示记录
4. 进入详情页查看记录
5. 数据保存在本地

第一版闭环稳定后，再接入地图、照片、标签和导出。

### 2.2 MVP 功能模块

| 模块 | MVP 状态 | 说明 |
| --- | --- | --- |
| 首页 | 必做 | 展示入口和基础统计 |
| 新增记录 | 必做 | 创建文字、时间、地点记录 |
| 时间轴 | 必做 | 按时间倒序展示记录 |
| 记录详情 | 必做 | 查看、编辑、删除记录 |
| 本地数据库 | 必做 | 使用 Room 保存结构化数据 |
| 地图 | 第二阶段 | 展示带经纬度的记录点 |
| 照片导入 | 第二阶段 | 使用 Android Photo Picker |
| 标签 | 第三阶段 | 支持记录分类和筛选 |
| 数据导出 | 第三阶段 | 导出 JSON，后续支持 Markdown/PDF |

### 2.3 第一版暂不做

- 登录注册
- 云同步
- 社交分享
- 情侣或家庭共同地图
- AI 总结
- 自动后台定位
- 多端同步
- Web 管理后台
- 复杂年度报告

这些能力会显著扩大工程复杂度，暂不纳入第一版。

## 3. 核心用户流程

### 3.1 新增记录流程

```text
打开 App -> 点击新增 -> 填写标题/正文/时间/地点 -> 保存 -> 返回时间轴 -> 看到新记录
```

### 3.2 查看记录流程

```text
打开时间轴 -> 点击记录卡片 -> 进入详情页 -> 查看完整内容
```

### 3.3 地图回看流程

```text
打开地图 -> 查看记录点 -> 点击 marker -> 查看记录摘要 -> 进入详情页
```

### 3.4 数据导出流程

```text
打开设置 -> 点击数据导出 -> 生成 lifeatlas_export.json -> 系统分享或保存文件
```

## 4. 页面规划

### 4.1 首页 Home

首页提供主入口和轻量统计：

- 总记录数
- 总照片数
- 去过的城市数量
- 最近一条记录
- 地图、时间轴、新增记录入口

### 4.2 时间轴 Timeline

按记录发生时间倒序展示：

- 年月分组
- 标题
- 地点
- 时间
- 正文摘要
- 首图缩略图，照片能力接入后展示

### 4.3 新增记录 AddRecord

字段：

- 标题
- 正文
- 发生时间
- 地点名称
- 经纬度，MVP 可先手动输入或留空
- 心情
- 重要程度
- 照片，第二阶段接入
- 标签，第三阶段接入

### 4.4 记录详情 RecordDetail

展示：

- 标题
- 正文
- 时间
- 地点
- 心情
- 重要程度
- 照片列表
- 标签

操作：

- 编辑
- 删除
- 查看地图位置

### 4.5 地图 Map

展示所有带经纬度的记录点：

- marker 展示记录位置
- 点击 marker 显示记录摘要
- 点击摘要进入详情页

### 4.6 设置 Settings

第一版设置项：

- 数据导出
- 图片存储策略，照片能力接入后展示
- 隐私与本地数据说明
- 关于岁迹

## 5. 验收标准

MVP 完成时必须满足：

1. 可以新增记录
2. 可以填写标题和正文
3. 可以设置记录时间
4. 可以填写地点名称
5. 数据写入本地数据库
6. 关闭 App 后数据不丢失
7. 时间轴能按时间倒序展示记录
8. 点击记录可进入详情页
9. 详情页展示完整记录
10. 可以删除记录
