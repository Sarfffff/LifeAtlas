# 岁迹 LifeAtlas 迭代路线图

## V0.1 本地记录闭环

目标：先让产品立起来。

范围：

- Android 项目初始化
- 底部导航
- Room 数据库
- 新增记录
- 时间轴展示
- 记录详情
- 删除记录

验收：

- 可以新增一条文字记录
- 可以保存到本地数据库
- 可以在时间轴看到记录
- 可以进入详情页查看
- 关闭 App 后数据不丢失

## V0.2 地图与照片

目标：让“人生地图”的核心体验出现。

范围：

- 高德地图 SDK 接入
- 地图 marker 展示记录点
- Android Photo Picker 接入
- 记录关联照片
- 详情页展示照片
- 时间轴展示首图

验收：

- 带经纬度的记录能出现在地图上
- 新增记录时可以选择照片
- 详情页可以看到照片

## V0.3 标签与筛选

目标：让记录更可整理。

范围：

- 标签创建
- 标签绑定
- 时间轴标签筛选
- 搜索
- 心情和重要程度优化

验收：

- 一条记录可绑定多个标签
- 可按标签筛选记录
- 可按关键词搜索标题和正文

## V0.4 数据导出与备份

目标：用户不被 App 绑定。

范围：

- JSON 导出
- JSON 导入
- 本地备份包
- 导出格式版本化

验收：

- 用户可以导出完整结构化数据
- 导出数据可以被重新导入

## V1.0 长期使用版

目标：达到可以长期自用和小范围公开测试的质量。

范围：

- 数据库 Migration 完整
- 错误处理完善
- 图片缩略图和压缩
- 私密记录锁
- 年度回顾基础版
- 自动化测试覆盖核心数据层

## V2.0 云同步与多端

目标：支持多设备和高阶能力。

范围：

- 登录注册
- 云同步
- 图片对象存储
- Web 查看
- AI 年度总结
- 家庭或情侣共同地图

注意：V2.0 才开始引入后端高并发设计。
