# 岁迹 LifeAtlas 迭代路线图

## V0.1 本地记录闭环

目标：先让产品立起来。

范围：

- Android 项目初始化
- 底部导航
- Room 数据库
- 新增记录
- 时间轴展示
- 记录详情
- 删除记录

验收：

- 可以新增一条文字记录
- 可以保存到本地数据库
- 可以在时间轴看到记录
- 可以进入详情页查看
- 关闭 App 后数据不丢失

## V0.2 地图与照片

目标：让“人生地图”的核心体验出现。

范围：

- 高德地图 SDK 接入
- 地图 marker 展示记录点
- Android Photo Picker 接入
- 记录关联照片
- 详情页展示照片
- 时间轴展示首图
- 图片缩略图缓存
- 图片压缩图缓存

验收：

- 带经纬度的记录能出现在地图上
- 新增记录时可以选择照片
- 详情页可以看到照片
- 详情页和时间轴优先加载缩略图，避免直接加载大图

## V0.2.1 照片存储增强

目标：在 V0.1 URI 引用方案稳定后，增强照片长期可用性和加载性能。

范围：

- 为已选照片生成 App 私有缩略图
- 可选生成压缩图
- 原始 URI、缩略图路径、压缩图路径同时保存
- 图片缓存清理策略
- 照片失效时的占位和重选提示

验收：

- 相册原图较大时，详情页仍能快速展示缩略图
- 原始 URI 不可用时，仍可显示已缓存缩略图
- 删除记录时同步清理关联缓存文件

## V0.3 标签与筛选

目标：让记录更可整理。

范围：

- 标签创建
- 标签绑定
- 时间轴标签筛选
- 搜索
- 心情和重要程度优化

验收：

- 一条记录可绑定多个标签
- 可按标签筛选记录
- 可按关键词搜索标题和正文

## V0.4 数据导出与备份

目标：用户不被 App 绑定。

范围：

- JSON 导出
- JSON 导入
- 本地备份包
- 导出格式版本化
- 备份包包含照片缓存文件

验收：

- 用户可以导出完整结构化数据
- 导出数据可以被重新导入
- 备份导出的记录可以携带可恢复的照片资源

## V1.0 长期使用版

目标：达到可以长期自用和小范围公开测试的质量。

范围：

- 数据库 Migration 完整
- 错误处理完善
- 图片缩略图和压缩
- 私密记录锁
- 年度回顾基础版
- 自动化测试覆盖核心数据层

## V2.0 云同步与多端

目标：支持多设备和高阶能力。

范围：

- 登录注册
- 云同步
- 图片对象存储
- Web 查看
- AI 年度总结
- 家庭或情侣共同地图

注意：V2.0 才开始引入后端高并发设计。
